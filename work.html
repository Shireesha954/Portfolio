Fig. 00 — Index

Selected work

Four projects spanning production web apps, design systems, and one-off installations. Each includes the problem, the approach, and where it landed.

Fig. A — Web app

Loom

Problem: Textile designers were hand-drafting weave patterns in spreadsheets, with no way to preview the finished cloth before it went on the loom.

Approach: Built a canvas-based simulator that renders warp and weft interactions in real time, exportable as a loom-ready draft file. Built with vanilla JavaScript and Canvas for performance on low-power studio machines.

Result: Adopted by two textile studios; cut sampling time for new patterns by roughly half, by their own estimate.

  • Canvas API
  • Vanilla JS
  • File export
Fig. B — Design system

Cartograph

Problem: A metro transit agency had three separate rider-facing apps, each with its own inconsistent — and largely inaccessible — map component.

Approach: Designed and led a shared React component library with keyboard-navigable maps, high-contrast route overlays, and screen-reader-friendly stop announcements, backed by an internal accessibility audit process.

Result: Rolled out across all three apps; the agency's next public audit reported full WCAG 2.1 AA conformance for the map components.

  • React
  • ARIA live regions
  • Design tokens
Fig. C — Installation

Tidepool

Problem: Wanted to explore whether ambient light could respond to a room's sound the way a tidepool responds to the tide — a personal studio brief, not a client one.

Approach: Built a Web MIDI controller paired with an Arduino-driven light rig, translating ambient audio input into slow, layered color shifts.

Result: Shown at a local maker fair over a single weekend; the piece ran unattended for eight hours a day without a single dropped frame.

  • Web MIDI
  • Arduino
  • Generative audio
Fig. D — Publishing tool

Fieldnotes

Problem: Independent researchers needed a lightweight way to publish field notes with maps and photos, without adopting a full CMS.

Approach: A static-site generator with a semantic HTML template library, built so every generated page passes an automated accessibility check before publishing.

Result: Used by a handful of independent researchers; one field journal built with it now has a small recurring readership.

  • Static site generation
  • Semantic HTML
  • Automated a11y checks
